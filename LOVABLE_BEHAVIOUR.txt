================================================================================
LOVABLE AI BEHAVIOUR GUIDELINES - FEINAUFMASS TECHNIKER APP
================================================================================

PROJEKT-UEBERSICHT
================================================================================
- App-Typ: Mobile-First Web-App für selbstständige Feinaufmaß-Techniker
- Zielgruppe: Techniker die Aufträge von Herstellern/Händlern annehmen
- Hauptfunktion: Aufträge empfangen, annehmen/ablehnen, als erledigt markieren
- Sprache: Deutsch (alle UI-Texte)

================================================================================
DATENMODELL & ENUMS
================================================================================

ORDER STATUS (Pflicht-ENUM):
- new       → Neuer Auftrag, noch nicht bearbeitet
- accepted  → Techniker hat Auftrag angenommen
- completed → Auftrag wurde erledigt
- rejected  → Techniker hat Auftrag abgelehnt

PROJECT TYPE (Pflicht-ENUM):
- Küche
- Schrank
- Bad
- Garderobe
- Wohnzimmer
- Sonstiges

REGEL: Status-Felder IMMER als TypeScript Union Types oder Supabase ENUMs.
       NIEMALS als freier String/varchar.

================================================================================
BUSINESS-LOGIK REGELN
================================================================================

AUFTRAGS-WORKFLOW:
1. Neue Aufträge erscheinen im "Aufträge"-Tab mit Status "new"
2. Techniker kann Auftrag ANNEHMEN → Status wird "accepted"
3. Techniker kann Auftrag ABLEHNEN → Status wird "rejected"
4. Nur angenommene Aufträge können als ERLEDIGT markiert werden → "completed"
5. Abgelehnte und erledigte Aufträge erscheinen im "Erledigt"-Tab

STATUS-UEBERGAENGE (erlaubt):
- new → accepted
- new → rejected
- accepted → completed
- VERBOTEN: completed → any, rejected → any

================================================================================
UI/UX PRINZIPIEN
================================================================================

MOBILE-FIRST:
- Optimiert für Smartphone-Nutzung
- Touch-freundliche Buttons (min. 44px)
- Bottom Navigation für Haupt-Tabs
- Keine Hover-States als einzige Interaktion

BOTTOM NAVIGATION (3 Tabs):
1. Aufträge - Neue und angenommene Aufträge
2. Erledigt - Abgeschlossene und abgelehnte Aufträge
3. Profil - Techniker-Einstellungen

STATUS-FARBEN (einheitlich):
- Neu (new):        Blau    → bg-blue-100, text-blue-700
- Angenommen:       Orange  → bg-orange-100, text-orange-700
- Erledigt:         Grün    → bg-green-100, text-green-700
- Abgelehnt:        Rot     → bg-red-100, text-red-700
